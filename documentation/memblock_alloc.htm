​LinuxKI Warning

 

Suspect SOS_BLOCKALLOCPARTIALLIST spinlock contention

05/06/2025       

 

Problem Description

 

Customer recently purchased a new HPE Compute Scale-up Server 3200 with 4 Intel® Platinum 8468H processors.   They migrated their Windows 2019 MS SQL Server environment from a physical Dell R740 server with 2 Intel® Gold 5220R processors (48 cores) to a 128-core VM running on the CSUS 3200.     The workload performs worse on the CSUS 3200 and the CPU usage is often reaching 100% busy.

 

Investigation

 

WinKI data (LinuxKI for Windows) was collected for analysis when the CPU usage was high.   The SQL Server threads were spending a lot of time in Spinlock code when allocating memory block.   The following data from kipid output for one of the SQL Server threads:

Resolution

 

This problem is documented in bug 2410400:

 

 SOS_BLOCKALLOCPARTIALLIST spinlock contention and elevated CPU utilization occur on large-memory machines.

 

After applying SQL Server 2019 Cumulative Update 21, the spinlock contention no longer existed performance exceeded the Dell server by 30%.
